Project Review – II

College Event Management System

Aditya University

Focus: Core Development Period: 20 – 28 February

Project Overview

A centralized web-based platform designed to digitally transform and streamline the management of college-level events.

  • Eliminates manual event handling, approvals, and registrations.
  • Provides dedicated, role-based interfaces for Admins, Organizers, and Students.
  • Ensures efficient event lifecycle management from initial creation to final participation.

Problem Statement

Current challenges in the existing manual ecosystem:

  • Manual event management causes significant delays, data inconsistency, and a lack of transparency.
  • Absence of a unified platform specifically built for event approvals, centralized registrations, and data analytics.
  • Administrative difficulty in tracking active participants, organizer efficiency, and overall event performance.

Project Objectives

1. Security

Implement robust, secure role-based authentication protocols.

2. Workflow Streamlining

Enable seamless event creation, hierarchical approval, and registration logic.

3. Visibility

Provide dynamic, real-time dashboards and actionable analytics.

4. Scalability

Design a scalable, modular, and maintainable system architecture.

System Architecture

  • Frontend: React.js (Vite, React Router DOM)
  • Backend: Node.js with Express.js REST APIs
  • Database: MongoDB Atlas with Mongoose ODM
  • Communication: JSON over HTTP

Design Principle: Decoupled frontend and backend for independent scalability and maintenance.

Client (React) ⬇ JSON HTTP ⬆ Server API (Node/Express) ⬇ Mongoose ODM ⬆ Database (MongoDB)

System Workflow (Demo)

  1. Initiation: Organizer drafts and submits a new event proposal.
  2. Queuing: Event registers as 'Pending' in the Admin approval dashboard.
  3. Authorization: Admin reviews details and executes the 'Approve' action.
  4. Broadcasting: Approved event materializes in the Student feed.
  5. Consumption: Students register; metrics instantly update across all administrative dashboards via real-time fetch.
